# 世界の見た目を面白くする計画

## 方針転換

「いろんなバイオームを記号で並べる」より、**少ない地形を一つひとつ魅力的に作り込む**ほうが世界として強い。

世界を構成する地形は **3種だけ**：

1. **草原** — 主役。広がりがあって、風で揺れる草と、白い石がカルスト台地のように点在する。
2. **森** — 木が密に生えている領域。草原との対比。
3. **町** — 人と建物が集まる領域。

山・荒野・湿地などは一旦やめる。今の `generateWorld` の枠組みに縛られず、**ゼロから組み直す前提**で考える。

---

## 主役：「カルスト草原」をどう作るか

これが世界の基調色になるので、ここが魅力的なら世界全体が魅力的になる。

### 構成要素

- **草** — 既存の `GrassField`（風包絡で波打つ三角形）。これは完成しており、強い。
- **白い石** — 新規追加。様々なサイズの白〜淡灰色の不定形ポリゴン or 円。Perlinノイズで「石が集まる場所」と「集まらない場所」のムラを作る。**草の配置ノイズと別シードにする**ことで、草の濃い場所に石があったりなかったりする。
- **微妙な地面色のムラ** — 一様な背景色ではなく、ごく薄い緑〜黄緑のグラデーション or Perlinベースのパッチで、見えない程度に下地を揺らす。これだけで「広がり」が一段上になる。

### 「綺麗に配置されている」感の作り方

ランダムに撒くだけだと汚くなる。カルスト感を出すコツ：

- **石を群（クラスタ）にする** — 1個ぽつんではなく、3〜7個が寄り添ったクラスタ単位で配置。クラスタ内では中心の石が大きく、周辺は小さい。
- **石同士は重ならない・近接しすぎない最小距離を持つ** — Poisson-disk サンプリング、または「配置済み石との距離が一定以上」で棄却。
- **大きな石は希少にする** — 大1：中3：小10 くらいの比率。大きい石はランドマークになる。
- **回転と歪みを少しだけ加える** — 完全な円ではなく、6〜8角形にランダムなノイズを乗せた不定形。各石ごとに固定（毎フレーム変えない）。

### 「広がり」感

- 草の placement ノイズ scale を**もっと大きいスケール**（= 模様を大きく）にして、草の濃い領域・薄い領域の波を世界規模で作る。今の `placementScale = 0.001` より小さく（例：0.0003）。
- 風包絡もスケールを大きく取って、世界の端から端へ風がゆっくり流れる感じに。

---

## 森

草原との**コントラスト**が森を森にする。

- 木 = 濃い緑の円（葉）+ 短い茶色の縦線（幹）。シンプルだが記号性が高い。
- 木は草原より**ずっと密**に配置する（草原がスカスカに見えるくらい）。
- 森の中は草を生やさない（または極端に薄く）。地面色を**少し濃く**して暗い場所感を出す。
- 森の境界は矩形フチではなく、**外周に向かって木が疎になる**グラデーションで自然に消す。

森と草原が「色の濃さ」と「記号の密度」で違って見えれば、矩形を意識させない。

---

## 町

町は「人工物が並ぶ場所」。これは既存の `Village.js`（建物配置）が機能している。改善点だけ：

- 町の矩形範囲内では**草も石も生やさない**（地面色を土色 or 石畳パターンに）。
- 町の**外周**に畑・柵を置くと、自然から人工への遷移が生まれる。
- 入口（道との接続点）に門の建物を置く。

優先度は低い。草原と森が良くなれば町も浮かび上がる。

---

## 実装順序

1. **草原の白い石** — 新規 `StoneField` コンポーネント（or `GrassField` に統合）。Perlinでクラスタ化、Poisson-diskで近接抑制、サイズ階層あり。
2. **草原の地面色ムラ** — Canvas に薄いグラデ or ノイズパッチを最下層で敷く。
3. **草の配置ノイズを大きくして広がりを出す** — `GrassField` の `placementScale` 調整。
4. **森を木の散布に置き換える** — `TreeField` コンポーネント。森エンティティ内のみに配置。森内は草を抑制。
5. **山・その他バイオームを削除** — `generateWorld` から `Mountain` と「荒野・湿地…」の name を消す。道の山迂回ロジックも不要に。
6. **町まわりの仕上げ** — 余裕があれば。

## 既存コードとの関係

「今の仕組みに乗っかる必要はない」とのことなので：

- `generateWorld.js` は**大胆に削っていい**。山ロジック・shortcut判定・MSTすべて、3種地形で必要な分だけ残す。
- `GrassField` は完成しているので**残す**。ただし配置スケールは調整。
- `Mountain.js` は削除候補。
- `Field.js` の `type` は `grassland | forest` のみに絞る。

新規コンポーネント：

- `StoneField` — 白石の散布（カルスト用）
- `TreeField` — 木の散布（森用）
- 必要なら `GroundTint` — 地面色ムラの下地

3つとも `GrassField` と同じパターン（Perlin + クラスタ + 自身で描画）で書けるので、CLAUDE.md の設計原則に素直に乗る。

## 最初の一手

**`StoneField` を作って草原に重ねる**のが一番早く「ワクワク」につながる。草原の上に白い石が点在するだけで、世界の格が上がる。これを見て次を判断するのがよさそう。
